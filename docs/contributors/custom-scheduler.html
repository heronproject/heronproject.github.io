Creating a Custom Scheduler

Heron is meant to be run only in large computing clusters. In order to run Heron in production, you'll need to set up a scheduler that is responsible for cluster management. Heron supports three schedulers out of the box: Mesos, Aurora (which runs on Mesos), and a local scheduler.

If you'd like to run Heron on a not-yet-supported system, such as YARN or Amazon ECS, you can create your own scheduler using Heron's Java scheduler API, as detailed in the sections below.

Here are two example implementations for your perusal:

  • Mesos scheduler
  • Aurora scheduler

Setup

Maven:

<dependency>
  <groupId>com.twitter.heron</groupId>
  <artifactId>scheduler</artifactId>
  <version>TODO</version>
</dependency>

Gradle:

dependencies {
  compile group: "com.twitter.heron", name: "scheduler", version: "TODO"
}

Interfaces

Creating a custom scheduler involves implementing each of the following Java interfaces:

Interface Role Examples
IConfig Parsing and loading of configuration for the scheduler Aurora, local, Mesos
ILauncher Defines how the scheduler is launched Aurora, local, Mesos
IRuntimeManager Handles runtime tasks such as activating topologies, killing topologies, etc. Aurora, local, Mesos
IScheduler Defines the scheduler object used to construct topologies Aurora, local, Mesos
IUploader Uploads the topology to a shared location that must be accessible by the runtime environment of the topology Aurora, local, Mesos
